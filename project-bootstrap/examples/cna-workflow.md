# Example — CNA Daily Chart Building with AI

**Input given:** "I want to start a project about CNA daily chart building with AI."

---

## 1. Project summary
- **Domain:** Work
- **Type:** Data visualization + AI workflow
- **Goal:** Build CNA daily charts faster and more consistently with AI assistance.
- **Final output:** A repeatable workflow + chart templates that turn daily data into publish-ready charts.

## 2. Recommended structure
```text
cna-daily-charts/
├─ project-brief.md
├─ workflow.md          # current process → target process
├─ data-sources.md      # where the daily numbers come from
├─ chart-templates.md   # standard chart types and specs
├─ prompts.md           # prompts to generate/check charts
├─ tasks.md
└─ evaluation.md
```

## 3. First 5 tasks
1. Map the **current** CNA daily chart process, end to end.
2. List the recurring chart types and their data sources.
3. Capture 3–5 example charts as templates (what good looks like).
4. Write prompts that turn raw daily data into a chart spec.
5. Run one full day's charts through the new workflow and time it.

## 4. Files to create
- `workflow.md` — current vs. target process
- `data-sources.md` — feeds, formats, refresh timing
- `chart-templates.md` — standard chart specs
- `prompts.md` — generation + review prompts
- `evaluation.md` — is the chart correct, fast, on-brand?

## 5. Evaluation checklist
- [ ] A daily chart can be produced faster than the old way.
- [ ] Charts are visually consistent with CNA style.
- [ ] Data is traceable to a source.
- [ ] A teammate could follow `workflow.md` unaided.
