# Example — Personal AI Learning (Bear, MCP, skills, harness)

**Input given:** "I want to get serious about learning AI tooling — MCP, skills, harness, Bear notes."

---

## 1. Project summary
- **Domain:** Personal
- **Type:** Learning + knowledge management
- **Goal:** Build durable, hands-on understanding of AI tooling I actually use.
- **Final output:** A learning log + a set of working examples (skills, MCP setups) I can reuse.

## 2. Recommended structure
```text
ai-learning/
├─ goals.md         # what "good" looks like in 3 months
├─ resources.md     # docs, links, courses, people
├─ log.md           # dated entries: what I tried, what I learned
├─ experiments/     # small working examples
├─ tasks.md
└─ evaluation.md
```

## 3. First 5 tasks
1. Write 3 concrete things you want to be able to *do* (not "learn about").
2. Collect the 5 best resources for each into `resources.md`.
3. Build one tiny working example (e.g. a minimal skill).
4. Start `log.md` — one dated entry per session.
5. Schedule a fixed weekly slot to keep momentum.

## 4. Files to create
- `goals.md` — outcomes, not topics
- `resources.md` — curated, not exhaustive
- `log.md` — the habit that compounds
- `experiments/` — proof you actually did it

## 5. Evaluation checklist
- [ ] Each goal is a thing I can *do*, demonstrably.
- [ ] The log has entries from more than one week (consistency).
- [ ] At least one reusable example exists in `experiments/`.
- [ ] I dropped resources I never used (curation, not hoarding).
