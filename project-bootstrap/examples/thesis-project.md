# Example — Thesis Paper Database

**Input given:** "I want to start my thesis database project."

---

## 1. Project summary
- **Domain:** School
- **Type:** Research + knowledge management
- **Goal:** Build a searchable database of papers that supports my thesis argument.
- **Final output:** An organized paper database + linked concepts and cases I can cite from.

## 2. Recommended structure
```text
thesis-database/
├─ project-brief.md
├─ research-question.md   # the question(s) driving the thesis
├─ paper-database.md      # sources: citation, summary, relevance
├─ concepts.md           # key ideas and definitions
├─ cases.md              # examples / case studies
├─ tasks.md
└─ evaluation.md
```

## 3. First 5 tasks
1. Write the research question(s) in one or two sentences.
2. Decide the fields for each paper (citation, summary, key claim, relevance, tags).
3. Add the first 10 papers you already know.
4. Extract recurring concepts into `concepts.md`.
5. Note 2–3 cases/examples that anchor the argument.

## 4. Files to create
- `research-question.md` — the spine of the thesis
- `paper-database.md` — one entry per source
- `concepts.md` — shared vocabulary
- `cases.md` — concrete examples

## 5. Evaluation checklist
- [ ] Every paper entry links back to the research question.
- [ ] I can find a source by concept or tag in seconds.
- [ ] Summaries are in my own words (not just abstracts).
- [ ] The database is enough to draft a literature review.
